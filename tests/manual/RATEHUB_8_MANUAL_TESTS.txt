================================================================================
  RATEHUB.8 — MANUAL TEST CASES
  Features: Employee Hiring Flow, EOTM/EOTY Visibility, Ended Employee Access Control
================================================================================

PREREQUISITE ACCOUNTS
---------------------
- System Admin account (role: system_admin)
- Company Admin account (role: company_admin) — owns a company (e.g. TechCorp)
- Employee account A (role: employee) — NOT yet employed at TechCorp
- Employee account B (role: employee) — already a current verified employee at TechCorp


================================================================================
FLOW 1: JOB POSTING → HIRE → EMPLOYEE SEES EVERYTHING
================================================================================

TC-1.1  Company Admin Creates a Job Position
  PRE:    Log in as Company Admin
  STEPS:  1. Go to Company Admin Dashboard
          2. Navigate to Job Positions tab
          3. Create a new job position (title, description, department)
          4. Publish the job position
  EXPECT: Job position appears in the list with status "open"

TC-1.2  Employee Applies to a Job
  PRE:    Log in as Employee A (not employed at TechCorp)
  STEPS:  1. Go to Employee Dashboard → Job Board tab
          2. Find the job posted by Company Admin in TC-1.1
          3. Click "Apply" and submit the application (upload CV if required)
  EXPECT: Application status shows "pending"
          Employee sees the application in "My Applications" section

TC-1.3  Company Admin Sends Hire Invite
  PRE:    Log in as Company Admin
  STEPS:  1. Go to Company Admin Dashboard → Job Positions tab
          2. Click into the job position from TC-1.1
          3. Find Employee A's application
          4. Click "Hire" / send hire invite
  EXPECT: Application status changes to "hire_invited" or equivalent

TC-1.4  Employee Accepts Hire Invite
  PRE:    Log in as Employee A
  STEPS:  1. Go to Employee Dashboard → Job Board tab
          2. Find the application with hire invite
          3. Click "Accept Hire"
  EXPECT: - Application status updates to "accepted" / "hired"
          - Employment is automatically created with:
              verification_status = 'approved'
              is_current = true
              source = 'job_application'
          - WITHOUT refreshing the page, the employee immediately sees:
              * Their new employment in the Employment tab
              * Company appears in Overview tab stats (Verified Employments count goes up)
              * "Give Peer Feedback" button is visible in Overview tab
              * "Give Feedback" button is visible in Feedback tab

TC-1.5  Employee Appears in Company Admin's Employee List
  PRE:    Log in as Company Admin
  STEPS:  1. Go to Company Admin Dashboard → Employees tab
  EXPECT: Employee A is listed as a current verified employee

FLOW 4: COMPANY ADMIN ENDS EMPLOYEE — ALL ACCESS REVOKED
================================================================================

TC-4.1  Company Admin Ends an Employee
  PRE:    Log in as Company Admin
  STEPS:  1. Go to Company Admin Dashboard → Employees tab
          2. Find Employee A
          3. Click "End Employment" (or equivalent action)
          4. Confirm the action
  EXPECT: Employee A's employment record is updated:
            is_current = false
            end_date is set
          Employee A disappears from the active employees list

TC-4.2  Ended Employee — Overview Tab Hides Feedback Action
  PRE:    Log in as Employee A (employment ended at TechCorp)
  STEPS:  1. Go to Employee Dashboard → Overview tab
          2. Check the quick actions section
  EXPECT: - "Give Peer Feedback" card is NOT visible
            (because no current approved employments remain)
          - "Verified Employments" stat shows 0 (for that company)

TC-4.3  Ended Employee — Feedback Tab Hides "Give Feedback" Button
  PRE:    Log in as Employee A (employment ended)
  STEPS:  1. Go to Employee Dashboard → Feedback tab
  EXPECT: - Past received feedback is still visible (read-only)
          - "Give Feedback" button is NOT visible
          - Employee cannot navigate to /feedback and submit
            (backend rejects: "You must be a current approved employee")

TC-4.4  Ended Employee — Cannot Vote in EOTM
  PRE:    - Log in as Employee A (employment ended at TechCorp)
          - There is an open EOTM event at TechCorp
  STEPS:  1. Go to Employee Dashboard → EOTM tab
          2. Try to find the TechCorp event
  EXPECT: - TechCorp does not appear in Employee A's EOTM company list
            (because approvedEmployments filters out is_current = false)
          - If employee somehow sends a direct API request to vote,
            backend rejects: "You must be a current verified employee to vote"

TC-4.5  Ended Employee — Cannot Vote in EOTY
  PRE:    - Log in as Employee A (employment ended at TechCorp)
          - There is an open EOTY event at TechCorp
  STEPS:  1. Go to Employee Dashboard → EOTY tab
          2. Try to find the TechCorp event
  EXPECT: - TechCorp does not appear in Employee A's EOTY company list
          - If employee somehow sends a direct API request to vote,
            backend rejects: "You must be a current verified employee to vote"

TC-4.6  Ended Employee — Not a Nominee in EOTM
  PRE:    - Employee A was ended from TechCorp
          - There is an open EOTM event at TechCorp
  STEPS:  1. Log in as Employee B (current employee at TechCorp)
          2. Go to Employee Dashboard → EOTM tab
          3. Open the TechCorp EOTM event
          4. View the nominee list
  EXPECT: Employee A does NOT appear in the nominee list
          (getEventNominees filters by is_current = true)

TC-4.7  Ended Employee — Not a Nominee in EOTY
  PRE:    - Employee A was ended from TechCorp
          - There is an open EOTY event at TechCorp
  STEPS:  1. Log in as Employee B (current employee at TechCorp)
          2. Go to Employee Dashboard → EOTY tab
          3. Open the TechCorp EOTY event
          4. View the nominee list
  EXPECT: Employee A does NOT appear in the nominee list

TC-4.8  Ended Employee — Cannot Submit Feedback via Direct API
  PRE:    Employee A is ended from TechCorp
  STEPS:  1. Send POST /api/feedback with Employee A's auth token
          2. Target a coworker at TechCorp
  EXPECT: API returns error:
          "You must be a current approved employee at this company to submit feedback"

EDGE CASES
================================================================================

TC-E.1  Employee with Multiple Employments — Ending One Doesn't Affect Others
  PRE:    Employee has verified employment at Company X AND Company Y
  STEPS:  1. Company X admin ends the employee
  EXPECT: - Employee can still give feedback at Company Y
          - Employee can still vote in EOTM/EOTY at Company Y
          - Employee cannot give feedback at Company X
          - Employee cannot vote in EOTM/EOTY at Company X

TC-E.2  Employee with No Employments — Clean State
  PRE:    New employee account with no employments
  STEPS:  1. Go to Employee Dashboard
  EXPECT: - Overview shows 0 Verified Employments
          - "Give Peer Feedback" quick action is NOT visible
          - Feedback tab: "Give Feedback" button is NOT visible
          - EOTM/EOTY tabs are visible but show no company events
          - Job Board tab is functional (can browse + apply)

TC-E.3  Rehire After Being Ended
  PRE:    Employee A was previously ended from TechCorp
  STEPS:  1. Company Admin posts a new job
          2. Employee A applies and gets hired again
          3. Employee A accepts hire invite
  EXPECT: - A new employment record is created (is_current = true)
          - Employee A regains full access: EOTM, EOTY, feedback
          - Employee A appears in CA Employees tab again
          - Employee A appears as a nominee in EOTM/EOTY events
