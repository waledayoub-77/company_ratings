================================================================================
ACCESS CONTROL & SECURITY
================================================================================

Vote Visibility:
  - OPEN Event + Non-Admin Employee: see "Vote counts hidden"
  - CLOSED Event + Any User: see actual vote counts
  - Event Admin/System Admin: always see vote counts

Certificate Access:
  - WINNER (user.id === certificate.user_id): CAN download certificate
  - NON-WINNER Employees: NO certificate button shown
  - Company Admins: NO certificate button shown or available
  - System Admins: Would need separate implementation

Vote Casting Restrictions:
  - Only verified/approved employees can vote
  - Cannot vote for themselves (system enforces)
  - One vote per employee per event

Event Creation:
  - Only current month/year allowed for EOTM
  - Only current year allowed for EOTY
  - Only company admin can create events

Event Closure:
  - Only company admin can close event
  - Triggers vote tally and winner determination
  - Non-winners get no Hall of Fame entry (NULL employee_id)

================================================================================
DATA FLOW
================================================================================

1. EOTM Event Creation Flow:
   Admin -> Create Event (current month only) -> Event OPEN
   -> All active employees added as candidates
   -> Employees vote -> Admin closes event -> Winner determined
   -> Entered to employee_of_month table -> Certificate ready

2. Vote Casting Flow:
   Employee -> View event -> See nominees (vote counts hidden)
   -> Cast vote for nominee -> API validates (not self, verified only)
   -> Vote recorded -> Confirmation notification

3. Event Closure Flow:
   Admin -> Close Event -> Backend tallies votes
   -> Winner determined (by vote count, then earliest vote timestamp for ties)
   -> Loser gets NULL employee_id in database
   -> Vote counts now visible to employees
   -> Winner can download certificate

4. Certificate Download Flow:
   Winner -> Views past winners -> Sees their name
   -> Clicks "Get Certificate" -> Modal opens
   -> Shows certificate preview -> Clicks "Download as Image"
   -> Canvas rendered to PNG -> Browser downloads PNG file

TECHNICAL DETAILS
================================================================================

Canvas Certificate Generation:
  - Canvas size: 1700x2200 pixels (8.5" x 11" letter at 200dpi)
  - Fonts: Georgia serif for elegant appearance
  - Colors:
    * Background: #fffbeb (amber-50)
    * Text: #1e293b (#78350f titles, #b45309 secondary)
    * Borders: #f59e0b (outer), #d97706 (inner)
    * Underlines: #f59e0b
  - Output: PNG image via canvas.toBlob()

Database Tables:
  - eotm_events: id, company_id, month, year, status, created_at, closed_at
  - eotm_votes: id, event_id, voter_id, nominee_id, created_at
  - employee_of_month: id, company_id, employee_id, event_id, month, year, votes_count
  - eoty_events: id, company_id, year, status, created_at, closed_at
  - eoty_votes: id, event_id, voter_id, candidate_id, created_at
  - employee_of_year: id, company_id, employee_id, event_id, year, votes_count

API Endpoints:
  EOTM:
    - POST /eotm/events (create event)
    - POST /eotm/events/:id/vote (cast vote)
    - PATCH /eotm/events/:id/close (close event)
    - GET /eotm/company/:companyId (get events)
    - GET /eotm/events/:id/nominees (get nominees with votes)
    - GET /eotm/company/:companyId/winners (get past winners)

  EOTY:
    - POST /eoty/events (create event)
    - POST /eoty/events/:id/vote (cast vote)
    - PATCH /eoty/events/:id/close (close event)
    - GET /eoty/company/:companyId (get events)
    - GET /eoty/events/:id/nominees (get nominees with votes)
    - GET /eoty/company/:companyId/winners (get past winners)
