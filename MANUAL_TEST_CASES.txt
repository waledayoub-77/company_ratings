================================================================================
  RE-TEST ROUND 1 - V2 Bug Fixes  (commit c76f600)
================================================================================

--------------------------------------------------------------------------------
R1-FIX 1 - Verification pending UI shown after submission  (was: V2-IV4/IV5)
--------------------------------------------------------------------------------
Root cause: After submitting, the form reloaded as a blank new form instead of
showing the pending state.
Fix: After submit, re-fetches status; isPending flag renders yellow pending block.

[ ] V2-F3-A  Login as employee  ->  /profile -> Verification tab
             ->  enter a valid URL (e.g. https://drive.google.com/testfile)
             ->  click "Submit for Verification"
             ->  form is REPLACED by yellow "Verification Pending" block
             ->  message: "Your verification request is being reviewed."


--------------------------------------------------------------------------------
R1-FIX 2 - Admin panel rejection notes work correctly  (was: V2-VA3/V2-F4-D)
--------------------------------------------------------------------------------
Root cause: rejectVerification API was not sending adminNotes to the backend.
Fix: API now sends { adminNotes } in request body; backend reads req.body.adminNotes.

[ ] V2-F4-D  Login as admin@ratehub.com  ->  /admin -> Verifications tab
             ->  find a pending request  ->  enter rejection notes
             ->  click "Reject"
             ->  request disappears from Pending  ->  appears under "Rejected" filter


--------------------------------------------------------------------------------
R1-FIX 3 - EOTM nominee names and vote counts correct  (was: V2-E5, V2-E7)
--------------------------------------------------------------------------------
Root cause: eotmService returned { employeeId, name, votes } but frontend
expected { employee_id, full_name, vote_count }.
Fix: renamed service fields to match frontend expectations.

[ ] V2-F6-B  Past winners in Hall of Fame show real employee names (not "Unknown")

[ ] V2-F6-C  Each nominee card shows correct vote count (e.g. "2 votes")
             and list is sorted highest-votes first


--------------------------------------------------------------------------------
R1-FIX 4 - Employee EOTM tab shows open events and allows voting  (was: V2-EV1-EV7)
--------------------------------------------------------------------------------
Root cause: getCompanyEvents returned rows with is_active boolean but
EmployeeDashboard filtered on ev.status === 'open' - nothing ever rendered.
Fix: service now maps status: ev.is_active ? 'open' : 'closed'.

[ ] V2-F7-C  Click "Vote" next to a nominee
             ->  button spins, nominee's vote count increments by 1

[ ] V2-F7-D  Vote a second time in the same event
             ->  inline error: "You have already voted..."
             ->  NO browser alert() dialog

[ ] V2-F7-E  Past winners section shows correct employee names and month/year labels

[ ] V2-F7-F  Company admin closes the event  ->  log back in as bob
             ->  closed event no longer in open events list
             ->  winner appears in "Past Winners" section
