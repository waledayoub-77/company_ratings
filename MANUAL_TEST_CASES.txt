R7-FIX 1 — Deleted user is fully blocked from login & email freed for re-use
           (was: BUG 1, R6-F5-B/C)
────────────────────────────────────────────────────────────────────────────────
Root causes (3 separate gaps):
  a) adminController.deleteUser set is_deleted + is_active but never set
     deleted_at — so authService.registerUser's .is('deleted_at', null) filter
     still found the old row and blocked re-registration.
  b) registerUser relied on deleted_at which was never set.
     Changed to .eq('is_deleted', false) which is the source of truth.
  c) refreshToken never checked user status — a deleted user could silently
     refresh their access token and keep using the app.
     Added is_deleted + is_active DB check inside refreshToken.

on the frontend, with no minimum length. Backend also only checked !reason.
