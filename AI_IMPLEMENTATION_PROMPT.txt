### FEATURE 1: Company Admin — Invite Employee

Company admin enters an employee's email address. System emails the employee
an invitation link with a unique token (UUID). Employee clicks the link:
  - If already registered → link them to the company (mark employment as approved)
  - If not registered → redirect to /register?inviteToken=... → auto-attach after registration

DB changes:
  ALTER TABLE employments ADD COLUMN source VARCHAR(20) DEFAULT 'request';
  ALTER TABLE employments ADD COLUMN invite_token VARCHAR(255);
  ALTER TABLE employments ADD COLUMN invite_expires_at TIMESTAMPTZ;
  ALTER TABLE employments ADD COLUMN invite_email VARCHAR(255);

Backend:
  POST /api/employments/invite — company_admin — body: { email, position, department, startDate }
    - Create employment row: verification_status='pending', source='invite', invite_token=uuid, invite_email=email
    - Send email with accept link
  POST /api/employments/accept-invite — public (no auth required initially)
    - body: { token }
    - Validate token, link user, set verification_status='approved'
  GET  /api/employments/pending-invites — company_admin — list pending invitations
  DELETE /api/employments/invite/:id — company_admin — cancel invite
  PATCH  /api/employments/invite/:id/resend — company_admin — regenerate token + resend

Frontend:
  CompanyAdminDashboard → Requests tab:
    - "Invite Employee" button → modal: Email, Position, Department, Start Date
    - Show pending invites list with Cancel / Resend actions
  New page /accept-invite?token=... :
    - Logged in: show company name + "Accept invitation" button
    - Not logged in: redirect to /register?inviteToken=...
  RegisterPage: if inviteToken in URL → call accept-invite after registration succeeds

### FEATURE 4: Company Email Locking

When employment ends at a company, the work email used there is locked for
that company — the employee cannot rejoin using the same work email.

DB changes:
  ALTER TABLE employments ADD COLUMN company_email VARCHAR(255);
  CREATE TABLE revoked_company_emails (
    id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    company_id   UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
    company_email VARCHAR(255) NOT NULL,
    employee_id  UUID NOT NULL REFERENCES employees(id),
    ended_at     TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(company_id, company_email)
  );

Backend:
  When employment ends (PATCH is_current=false or end-by-admin):
    - If company_email is set → INSERT into revoked_company_emails
  When new employment is created or invited:
    - Check revoked_company_emails for (company_id, company_email) match → 403 if found

Frontend:
  Employment request / invite acceptance form:
    - Optional "Work Email" field (the company-issued email)
  CompanyAdminDashboard → approval view:
    - Show the company_email if provided, allow admin to enter/edit before approval
